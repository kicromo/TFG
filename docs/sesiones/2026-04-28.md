# Sesión 2026-04-28 — Configuración del entorno y exploración del hardware

**Objetivo**: Primer acceso al RPi5, montaje del sistema de archivos remoto, y verificación de las capacidades reales de la cámara C270.

**Hardware**: RPi5 (Debian 13 Trixie, 117 GB libres) + Logitech C270 HD Webcam (USB)
**IP RPi5 en LAN**: `192.168.1.38`
**IP PC en LAN**: `192.168.1.50`

---

## 1. Acceso SSH y montaje SSHFS

Para evitar transferir archivos con `scp` en cada edición, se monta el sistema de archivos del RPi5 en el PC mediante SSHFS.

**Máquina**: PC (España)

```bash
mkdir -p ~/rpi
sshfs pi_damh@192.168.1.38:/ ~/rpi
```

A partir de aquí, editar archivos en `~/rpi/home/pi_damh/` es equivalente a editarlos directamente en el RPi5.

Para desmontar cuando la RPi5 se desconecta:

```bash
fusermount -u ~/rpi
```

## 2. Identificación de dispositivos de vídeo

**Máquina**: RPi5

```bash
ls /dev/video*
```

```
/dev/video0   /dev/video21  /dev/video25  /dev/video29  /dev/video33
/dev/video1   /dev/video22  /dev/video26  /dev/video30  /dev/video34
/dev/video19  /dev/video23  /dev/video27  /dev/video31  /dev/video35
/dev/video20  /dev/video24  /dev/video28  /dev/video32
```

```bash
v4l2-ctl --list-devices
```

```
pispbe (platform:1000880000.pisp_be):
    /dev/video20 … /dev/video35
    /dev/media0, /dev/media1

rpi-hevc-dec (platform:rpi-hevc-dec):
    /dev/video19
    /dev/media2

C270 HD WEBCAM (usb-xhci-hcd.1-2):
    /dev/video0
    /dev/video1
    /dev/media3
```

**Análisis**:
- La WebCam USB es `/dev/video0`.
- `/dev/video20`–`/dev/video35`: nodos del procesador de imagen PiSP Backend del RPi5 (no son encoders H.264).
- `/dev/video19`: decodificador HEVC hardware (no encoder).

## 3. Capacidades de la cámara C270

**Máquina**: RPi5

```bash
v4l2-ctl --device=/dev/video0 --list-formats-ext
```

Resumen relevante:

| Formato | Resolución | FPS máx. |
|---------|-----------|----------|
| YUYV (raw) | 1280x720 | 7.5 fps |
| YUYV (raw) | 640x480 | 30 fps |
| MJPG (comprimido) | 1280x720 | 30 fps |
| MJPG (comprimido) | 1280x960 | 30 fps |

**Conclusión**: Para 720p a una tasa útil es obligatorio usar MJPG. Con YUYV a 720p el bus USB solo aguanta 7.5 fps porque el volumen de datos sin comprimir supera la capacidad del USB 2.0.
