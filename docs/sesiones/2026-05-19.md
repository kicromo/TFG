# Sesión 2026-05-19 — Implementación de capturer.py y corrección del GOP

**Objetivo**: Implementar el módulo de captura con segmentación automática, y resolver el problema de duración incorrecta de los segmentos.

**Máquina**: RPi5 (192.168.1.38)
**Fichero principal**: `nodo-remoto/capture/capturer.py`

---

## 1. Implementación del capturer.py

Se implementa un wrapper sobre FFmpeg que:
1. Lanza FFmpeg con segmentación de 3 s en el directorio `staging/`.
2. Monitoriza `staging/` en un hilo separado (polling cada 0.5 s).
3. Cuando hay dos o más archivos en `staging/`, todos excepto el último son segmentos completos: se mueven a `ready/`.
4. Al detenerse, mueve los parciales restantes de `staging/` a `ready/`.

**Comando FFmpeg generado por `build_ffmpeg_cmd()`**:

```bash
ffmpeg -hide_banner -loglevel warning \
  -f v4l2 -input_format mjpeg -video_size 1280x720 -framerate 15 -i /dev/video0 \
  -c:v libx264 -preset ultrafast -tune zerolatency -b:v 1000k \
  -f segment -segment_time 3 -segment_format mpegts -reset_timestamps 1 -strftime 1 \
  /var/camera-buffer/staging/seg_%Y%m%d_%H%M%S.ts
```

**Primer test de captura** (salida observada):

```
[INFO] Segmento listo: seg_20260519_101523.ts (638 KB)
[INFO] Segmento listo: seg_20260519_101540.ts (641 KB)
[INFO] Segmento listo: seg_20260519_101557.ts (637 KB)
```

**Problema detectado**: Los segmentos duran 17 segundos en lugar de 3.

---

## 2. Diagnóstico: problema de GOP (Group of Pictures)

FFmpeg solo puede cortar un segmento MPEG-TS en un fotograma clave (keyframe). El codificador `libx264` por defecto coloca un keyframe cada 250 fotogramas.

A 15 fps:

```
250 fotogramas / 15 fps = 16.7 segundos entre keyframes
```

FFmpeg no puede insertar el corte en el segundo 3 si el siguiente keyframe no llega hasta el segundo 17. El resultado son segmentos de ~17 s en lugar de 3 s.

**Solución**: forzar un keyframe exactamente cada 45 fotogramas (= 15 fps × 3 s), con los parámetros `-g 45 -keyint_min 45`.

---

## 3. Corrección del GOP y verificación

**Parámetros añadidos al comando FFmpeg**:

```
-g 45 -keyint_min 45
```

**Resultado tras la corrección**:

```
[INFO] Segmento listo: seg_20260519_103746.ts (384 KB)
[INFO] Segmento listo: seg_20260519_103749.ts (382 KB)
[INFO] Segmento listo: seg_20260519_103752.ts (388 KB)
[INFO] Segmento listo: seg_20260519_103755.ts (385 KB)
```

Los segmentos se producen cada 3 segundos con un tamaño de aproximadamente 384 KB, lo que corresponde al bitrate de 1 Mbps configurado (384 KB × 8 = 3072 kb / 3 s ≈ 1024 kb/s).

El directorio `ready/` acumula segmentos de 3 s listos para ser enviados por el módulo `sender` (pendiente de implementar).
